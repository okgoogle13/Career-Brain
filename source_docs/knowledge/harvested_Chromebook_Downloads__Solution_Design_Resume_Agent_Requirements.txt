# **Consolidated Solution Design: AI Career Co-Pilot (Firebase-Native)**

This document consolidates the project requirements for a personal AI career co-pilot, integrating the user's design specifications with the provided solution design template and aligning with Firebase-native best practices.

### **1\. Project Goal & Overview**

* **Objective:** Build a personal AI agent that provides a centralized, intuitive dashboard to automate the creation of tailored resumes, cover letters, and key selection criteria (KSC) responses for job applications in the Australian Community Services sector. The agent will also automate job scouting and calendar reminders, acting as a complete, all-in-one career co-pilot.  
* **Primary Users:** Job-seekers in the Australian Community Services sector.

### **2\. Core Features & Functional Modules**

* **Single-Page PWA Dashboard:** A card-based layout for all features, served via **Firebase Hosting**.  
* **One-Time Profile Setup:** The agent will learn the user's professional history from uploaded documents (resumes, cover letters) via a populate\_profile.py script.  
* **Document Generation:**  
  * Extracts job description and selection criteria directly from a provided URL.  
  * Generates tailored resumes, cover letters, and KSC responses using a sector-specific knowledge base and the user's pre-loaded profile.  
  * Allows the user to select a visual theme for PDF outputs and specify a professional tone of voice.  
* **ATS Analysis & Reporting:**  
  * Analyzes a generated resume against a job description to provide an ATS compliance score.  
  * Displays a list of matched/missing keywords and actionable suggestions in the UI.  
  * Provides a Best Practices Checklist in the UI.  
* **Automated Job Scouting:** A background **Cloud Function** will check the user's Gmail for new job alerts and display them on the dashboard.  
* **Automated Reminders:** Automatically creates events in the user's Google Calendar for application deadlines found in job ads.

### **3\. Inputs & Outputs**

#### **Input Specifications**

| Input Type | Format | Source | Parsing Tool | Mandatory? |
| :---- | :---- | :---- | :---- | :---- |
| **Historical Documents** | PDF, DOCX | User Upload (one-time) | pypdf, python-docx | Yes (for setup) |
| **Google Permissions** | OAuth | User (one-time) | Google API Client | Yes (for setup) |
| **Core Knowledge Artifacts** | .md, .json | Provided | Custom Parser | Yes (for setup) |
| **Job Advertisement URL** | URL | User (runtime) | requests, BeautifulSoup | Yes |

| **Selected PDF Theme ID** | String | User (runtime) | N/A | Yes |
| **Tone of Voice** | String | User (runtime) | N/A | Optional |
| **Gemini API Key** | String | User (.env file) | N/A | Yes |

#### **Desired Outputs**

* **Generated Documents:** Resume, cover letter, and KSC response (saved as DOCX and PDF).  
* **ATS Compliance Report:** Score and detailed analysis displayed in the UI.  
* **Best Practices Checklist:** Displayed in the UI.  
* **Job Opportunities:** New listings displayed on the dashboard.  
* **Calendar Events:** New application deadlines created in the user's Google Calendar.

### **4\. Workflow Steps**

1. **First-Time Use (Setup):** The user is guided through a one-time setup to (a) run the populate\_profile.py script to upload their career documents and build their profile in the **Firestore** database, and (b) grant Google permissions for Gmail/Calendar.  
2. **Standard Use (Generation):** The user pastes a job ad URL into the "Interactive Workflow" card, selects a theme, and clicks "Generate." The card updates to show the ATS analysis and provides download links for the generated documents.  
3. **Passive Use (Automation):** In the background, a scheduled **Cloud Function** periodically checks for new jobs, which appear on the dashboard, and creates calendar events for deadlines.

### **5\. Deployment & Technical Specifications**

* **Deployment:** Continuous Deployment from a GitHub repository using **GitHub Actions**. On push to the main branch, the workflow will automatically deploy updates to Firebase.  
* **Architecture:** A serverless, Firebase-native application:  
  * **Backend:** **Cloud Functions for Firebase (Python)** to handle API requests and business logic.  
  * **Scheduler:** **Cloud Functions** triggered by **Cloud Scheduler** (replaces Celery).  
  * **Frontend:** Progressive Web App (PWA) with basic HTML/JS, deployed to **Firebase Hosting**.  
* **Database:** **Cloud Firestore** for scalable, persistent user profile data (replaces SQLite).  
* **Authentication:** No login or complex user management required (single-user focus).  
* **Libraries:**  
  * pypdf, python-docx (Document Parsing)  
  * requests, BeautifulSoup (Web Scraping)  
  * google-api-python-client (Google Integration)  
  * firebase-admin (Python SDK for Firebase)  
* **Deliverables:**  
  * All code and assets structured for Firebase deployment.  
  * openapi.yaml file.  
  * .github/workflows/deploy.yml file for the GitHub Actions pipeline.  
  * populate\_profile.py script for profile setup.  
  * Comprehensive README.md with setup and deployment instructions.

### **6\. Prompt & Context Strategy**

* **System Prompt:** Will leverage a persona (e.g., "expert career coach for the Australian Community Services sector").  
* **Context Engineering:** The generation process **must** leverage the provided knowledge artifacts (RAG) and the pre-loaded user profile from the **Firestore** database to ensure high-quality, tailored outputs.

### **8\. Scope & Exclusions**

* **In Scope:**  
  * Set up a Firebase-native architecture with Cloud Functions and Firestore.  
  * Implement all document generation, parsing, web scraping, ATS analysis, and PDF/DOCX creation.  
  * Implement background job scouting and Google Calendar integration using Cloud Functions.  
  * Incorporate provided knowledge artifacts into the generation process (RAG).  
  * Provide the populate\_profile.py setup script.  
  * Create the complete PWA frontend with a card-based dashboard UI.  
  * Configure a GitHub Actions workflow for automated deployment.  
* **Out of Scope:**  
  * Building a native mobile app.  
  * Implementing user authentication or multi-user logic.  
  * Final deployment to a cloud provider (instructions will be provided).  
  * Managing 3rd-party API quotas.
